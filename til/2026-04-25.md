## 오늘 다룬 범위

### Week 1 — 선형대수 (D1~D5)
- **D1 벡터·내적**: 벡터의 두 얼굴(기하/대수), 길이 = √(Σvᵢ²), 내적의 두 정의 `Σaᵢbᵢ = |a||b|cosθ`, 의미 = 방향 유사도 점수
- **D2 행렬·벡터 곱**: 행 시야(내적 반복) + 열 시야(가중합) — 후자가 Attention 출력 모양과 동일
- **D3 행렬·행렬 곱**: 시야 1(내적 격자) + 시야 2(열 시야), shape 규칙 `(m,k)@(k,n)=(m,n)`, 전치 → `Q·K^T` 의 attention score matrix `(N,N)`
- **D4 행렬 = 공간 변형**: 행렬의 열 = 기저 벡터의 도착지. 변환 4종(스케일·회전·셰어·투영). 행렬곱 = 변환 합성. 비가환성.
- **D5 affine + 활성화**: `y = Wx + b` (변형 + 평행이동). 활성화 함수 없이는 깊이가 무의미함을 합성으로 증명.

### 보너스 — 직사각·고유분해·SVD
- **직사각 행렬**: `(m,k)` = "k차원 → m차원 변환기". 열공간(column space) 직관.
- **고유벡터·고유값**: `A·v = λ·v` 의 의미와 유래("A 를 반복 적용해도 살아남는 방향이 있는가?"). 손계산 절차 (`det(A-λI)=0` → λ → v).
- **대각화**: `A = PDP⁻¹`. 좋은 좌표계로 옮기면 단순 스케일링.
- **SVD**: `A = UΣV^T`. 모든 행렬에 대해 "회전→스케일→회전" 3단 분해.

### Week 2 — 미적분 (D1~D2 일부)
- **D1 미분의 의미**: 접선 기울기 = 변화율. ML 에선 손실 함수의 내리막 방향을 알기 위한 도구. `f(x)=x²-4x+5` 의 최솟값 x=2 직접 풀이.
- **Gradient descent 시뮬레이션**: `x_new = x_old - η·f'(x_old)`. 손계산 + NumPy 코드 양쪽으로.
- **D2 편미분·그래디언트**: 다변수 함수에서 한 축씩 기울기. `∇f` = 편미분의 벡터, "가장 가파른 오르막 방향". `f(x,y)=3x²+2y³` 의 `∇f(1,2) = [6, 24]` 직접 계산.
- **그래디언트 활용**: 다변수 경사하강법 시뮬레이션 (`f(x,y)=x²+y²` 시작점 (3,4) → (0,0) 수렴). 학습률의 영향(작으면 느림, 크면 발산).

## 핵심 3줄 (오늘 가장 강하게 박힌 통찰)

1. 오랜만에 기하학 기초를 다시 생각해보는 시간을 가졌다,,,멈췄던 머리가 돌아가기 시작한다.
2. **행렬은 단위 벡터(기저)가 어디로 갈지를 정해주는 명세서다.** shape `(m,k)` 는 "k차원 입력 → m차원 출력" 변환기이고, 행렬곱은 변환의 합성이라 순서가 중요. 이 시야 하나로 Attention `Q·K^T` 가 attention score matrix 가 되는 이유, 신경망 layer width 의 의미가 다 잡힌다.
3. **고유벡터/대각화/SVD는 모두 "행렬의 본질을 드러내는 좋은 좌표계 찾기" 라는 한 주제의 변주다.** 고유벡터는 A 가 반복 적용돼도 방향이 보존되는 특별한 축. 그 축들로 좌표계를 갈아끼우면 복잡한 변환이 단순 대각 스케일링으로 변신. SVD 는 이 아이디어를 모든 행렬에 일반화한 것 (회전→스케일→회전).

## 막힌 것 / 궁금한 것
- 오랜만에 행렬곱의 의미를 다시 생각해보니, 정말 헷갈리는 포인트가 많다. 머리를 다시 굴리는데 시간이 필요할꺼 같다,,,,
- 고유벡터를 처음엔 "A 의 열에 직접 들어있는 값" 으로 오해했다. A 의 열(=기저 도착지)과 A 의 고유벡터(=A 가 반복 적용돼도 방향 보존되는 특별 벡터)는 다른 종류의 정보임을 깨달았다.
- 행렬 곱과 스칼라 곱의 차이가 처음엔 흐릿했다. 스칼라 곱은 같은 직선 위에서 길이 조정(음수면 반대 방향)만 가능하지만, 행렬 곱은 임의 방향으로 회전·꺾임 가능. 고유벡터는 "행렬 곱인데 결과가 스칼라 곱처럼 보이는" 특별한 입력.
