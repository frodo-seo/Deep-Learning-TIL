## 오늘 다룬 범위

### Week 2 D3~D5 — 연쇄법칙, Softmax+CE, MLE/KL
- **연쇄법칙**: `dy/dx = dy/dg · dg/dx`. 역전파 = 연쇄법칙으로 모든 파라미터의 기울기를 구하는 과정.
- **Softmax+CE**: logit → 확률 → `-log(정답 확률)`. 그라디언트 = `ŷ - y` (그냥 빼기).
- **CE = MLE = KL 최소화**: 전부 같은 말. CE가 표준인 이유는 통계적 최적(MLE)을 수식화한 것.
- 손실함수는 non-convex → local minimum. 기울기 0은 "목표가 아니라 결과".

### Week 3 — PyTorch & autograd
- `torch.Tensor`: NumPy + autograd + GPU. `requires_grad=True`가 역전파 스위치.
- 학습 루프 4줄: `zero_grad → forward → backward → step`
- `nn.Linear`로 선형회귀: 랜덤 w,b에서 시작 → 학습으로 정답 발견.
- Adam: 파라미터마다 학습률 자동 조절 (SGD보다 세심, 메모리 3배).
- `w_new = w_old - lr * grad`: 기울기의 반대(내리막)로 이동.

### Week 4 — micrograd
- `Value` 클래스: 매 연산마다 출처를 기록 → backward 때 거꾸로 타고가며 기울기 전파.
- 덧셈 노드: 기울기 그대로 전달. 곱셈 노드: 상대방 값을 곱해서 전달.
- 직접 만든 엔진으로 `y=3x+1` 학습 성공 → PyTorch와 결과 일치 확인.

### Week 5 — 최적화, 초기화, 정규화
- **Adam**: Momentum + 학습률 적응. 실전 표준.
- **Xavier 초기화**: 입력 많을수록 w를 작게 → 값 폭발/소실 방지. w=0이면 대칭성 문제로 학습 불가.
- **LayerNorm**: 토큰 하나의 임베딩 벡터(C축) 안에서 정규화. Transformer 표준.
- **Dropout**: 랜덤으로 뉴런 꺼서 과적합 방지.

### Week 6 — makemore 언어모델
- **Bigram**: 카운트 기반 vs 신경망 기반 — 같은 확률표를 학습으로 알아냄.
- **Embedding**: 원핫 @ W = W에서 해당 행을 꺼내는 것 = 룩업 테이블.
- **MLP 언어모델**: 문맥 3글자 → 임베딩 → 이어붙이기 → Linear → 예측.

### Week 7 — Attention 수식 완전정복
- **Q, K, V**: 구조가 역할을 강제함. 사람은 구조를 설계, 값은 모델이 학습.
  - Q·K^T = 유사도, 높을수록 V가 크게 반영.
  - Q,K,V의 정답은 없음 — "다음 토큰 맞추기" 하나로 전부 학습됨.
- **Scaled**: `QK^T / √d_k`. 차원 커지면 분산 커짐 → 나눠서 softmax가 부드럽게.
- **Causal Mask**: 미래 토큰 점수를 -inf → softmax 후 0. Decoder(GPT)에만 적용.
- **Multi-Head**: 512차원을 8×64로 쪼개 → 각 Head가 다른 관점(의미/문법/위치)으로 Attention.
- **복잡도**: O(T²·d). 토큰 수 제곱에 비례 — Transformer 최대 약점.

### Week 8 — Transformer 블록
- **Positional Encoding**: 토큰 임베딩 + 위치 임베딩. 없으면 순서 구분 불가.
- **FFN**: 2층 신경망 (512→2048→512). Attention은 토큰 간, FFN은 토큰 내 가공.
- **Residual**: 변환 전 값을 더해줌 → 기울기 지름길 → 깊이 쌓아도 학습 가능.
- **Pre-LN**: 변환 전에 정규화 → Post-LN보다 안정적. GPT 표준.
- **Decoder-only (GPT)**: 토큰화 → 임베딩+위치 → 블록×N → 마지막 토큰 → Linear → softmax → 다음 토큰.

### Week 9~10 — nanoGPT 학습

- 3000 step 후 "KING RICHARD", "thy", "shall" 등 Shakespeare 스타일 텍스트 생성 성공.

## 보너스 트랙 — Frontier 모델로 가는 다리

### Frontier 모델은 nanoGPT와 뭐가 다른가
- **골격은 동일하다**: decoder-only Transformer + causal attention + next-token prediction. GPT-4, Claude, Gemini, Llama, DeepSeek 전부 같은 뼈대.
- 성능 향상은 5축으로 분리된다:
  1. **Scale**: nanoGPT ~10M → frontier ~1T+ params, ~15T+ tokens. Chinchilla 법칙(params와 data를 같이 늘려야).
  2. **아키텍처 부품 교체**: LayerNorm→**RMSNorm**, 절대 PE→**RoPE**, GELU→**SwiGLU**, MHA→**GQA**, **Flash Attention**, **MoE**.
  3. **데이터 품질**: 양보다 질. 필터링/중복제거/합성 데이터. Phi 시리즈가 증명.
  4. **Post-training**: SFT → RLHF/DPO → Constitutional AI → RLVR. nanoGPT는 pretraining까지만.
  5. **Test-time compute**: o1, Claude thinking. 추론 시점에 더 오래 생각하게 = 새 scaling 축.

### DeepSeek V4 (2026-04-24 공개) 뜯어보기
- V4-Pro: **1.6T total / 49B active**. V4-Flash: 284B / 13B. MIT 라이선스.
- V3.2 대비 1M 컨텍스트에서 **FLOPs 27%, KV cache 10%** 만 사용.
- 핵심 3가지:
  - **Hybrid Attention (CSA + HCA)**: K,V를 시퀀스 축으로 압축 + top-k 선택.
    - CSA: m개 토큰을 1개로 압축 → Lightning Indexer로 top-k 추림 + sliding window.
    - HCA: 더 공격적 압축 (m'≫m), dense attention.
    - CSA/HCA 레이어 인터리빙으로 1M 컨텍스트 가능.
  - **mHC (Manifold-Constrained Hyper-Connections)**: residual 매핑 행렬을 doubly stochastic manifold에 제약 → spectral norm ≤ 1 → 깊은 망에서 forward/backward 수치 안정.
  - **Muon Optimizer**: AdamW 대체. gradient를 직교화해 update 방향 간 상관성 제거. 1.6T 모델 학습 안정화.
- 정리: "성능을 올렸다"보다 **"같은 성능을 훨씬 싸게, 훨씬 긴 컨텍스트에서"** 가 정확.

### KV 압축의 계보 — 저랭크 분해의 진화
- 출발 직관: Q·K^T가 비싸니 K,V를 분해해서 중요한 것만 남기자 (= Week 1 SVD의 발상).
- **그냥 SVD로는 안 되는 이유**:
  - K,V는 weight가 아니라 입력마다 달라지는 *activation* — 미리 분해 불가.
  - SVD의 분해 기준은 reconstruction error → downstream loss와 무관.
  - 해법: **학습 가능한 low-rank** (분해 행렬 자체를 학습).

| 방법 | 분해 대상 | 분해 방식 |
|------|---------|---------|
| LoRA | weight update ΔW | $\Delta W = BA$, rank r |
| Linformer (2020) | K, V (sequence 축) | 학습된 projection |
| **MLA** (DeepSeek V2/V3) | K, V (feature 축) | 학습된 low-rank 분해 |
| **CSA** (DeepSeek V4) | K, V (sequence 축) | 학습된 compression + top-k |

### Attention 식, 손볼 수 있는 4자리

$$\text{Attn}(Q,K,V) = \text{softmax}\!\left(\frac{QK^T}{\sqrt{d}} \odot M\right) V$$

| 손보는 곳 | 대표 기법 | 아이디어 |
|---------|---------|---------|
| **M (마스크)** | Longformer, BigBird, NSA, **V4 CSA top-k** | 어떤 (i,j) 쌍만 계산할지 |
| **K, V (활성화)** | Linformer, **MLA** | 작게 압축한 뒤 attention |
| **연산 자체** | Linear Attention, Performer | softmax를 kernel로 근사 → O(T) |
| **Q·K^T 자체 안 함** | RWKV, Mamba (SSM) | attention 버리고 다른 시퀀스 모델 |

- V4 = **M + K,V 둘 다** 손보는 하이브리드.
- 마스크는 정적(causal/window) vs 동적(top-k) 두 종류. V4 top-k는 **동적** — 모델이 "지금 뭘 처리하려면 어디를 봐야 하지"를 스스로 학습.

### 보너스의 보너스 — 프롬의 *소유 vs 존재* 와 LLM 기억
- 에리히 프롬 *소유냐 존재냐* (1976): 인간이 세계와 관계 맺는 두 양식.
  - **소유 양식**: 자아 = 가진 것. 기억 = 저장+인출. 외부 메모/사진/주소록.
  - **존재 양식**: 자아 = 살아가는 방식. 기억 = 능동적 재구성, 살아있고 변형됨.
- LLM의 두 기억층 매핑이 깔끔하게 떨어진다:
  - **Weights = being-mode**: 학습 데이터를 *겪고* 변한 결과. 분리 인출 불가, 통합돼 있음.
  - **Context / KV cache = having-mode**: 토큰 그대로 저장, 인덱스로 인출, 세션 끝나면 반납.
  - 정확한 한 줄: **존재 양식으로 형성된 가중치를 통해, 소유 양식으로 보관된 컨텍스트를 처리한다**.
- 추론 시점의 LLM은 변하지 않음(가중치 동결) → being이 구조적으로 불가능.
- *being memory*가 살아있다는 건 세 가지 관계가 동시에 작동한다는 뜻:
  1. 나와 기억 대상 (통합 vs 분리)
  2. 기억과 기억 (연결망 vs 단절된 항목)
  3. 과거와 현재 (살아 움직임 vs 정지)
- 핵심: 명사("기억을 *가진다*") → 동사("*기억하고 있다*")의 전환이 두 양식의 차이.
- 공학적 함의: continual learning, RAG, fine-tuning, TTT, model editing 모두 사실상 "LLM에 *being memory*를 주려는 시도"의 변형들.

## 핵심 3줄

1. **Attention의 Q,K,V는 정답이 없다.** 사람은 "내적으로 비교하고 가중합하라"는 구조만 설계했고, 모델이 "다음 토큰 맞추기"라는 목표 하나로 알아서 역할을 학습한다.
2. **Transformer = 이미 배운 부품들의 조합.** 행렬곱(Week1) + 역전파(Week2) + Softmax+CE(Week2) + Attention(Week7) + FFN + Residual + LayerNorm.
3. **랜덤 가중치에서 시작해 Shakespeare를 생성하기까지**, 원리는 하나: forward → loss → backward → update 반복.
